# Plan 02: Audio Capture & Engine Pipeline

**Phase:** 01 — Audio-reactive LED effects
**Complexity:** High
**Depends on:** Plan 01 (needs `BandAmplitudes`, `AudioCmd`, `LedOutput`, `AudioEffect`, `AudioEffectState`, `fft::process_fft`, `fft::smooth_bands`)
**Wave:** 2

## Objective

Implement the audio capture layer (cpal device enumeration + input stream) and the complete audio engine pipeline: audio callback → FFT → smoothing → effect render → LED output. This plan creates the runtime threads that drive LEDs from audio data.

## Context

- **Files from Plan 01 that this plan depends on (read these first):**
  - `aucc-rs/src/audio/mod.rs` — `BandAmplitudes`, `AudioCmd`, `LedOutput`
  - `aucc-rs/src/audio/fft.rs` — `process_fft()`, `smooth_bands()`
  - `aucc-rs/src/audio/effects.rs` — `AudioEffect`, `AudioEffectState`

- **cpal API patterns (from RESEARCH.md):**
  - `cpal::default_host()` → ALSA host on Linux
  - `host.devices()` → iterator of all audio devices (needs `DeviceTrait` import)
  - `device.name()` → device name string (use for matching + display)
  - `device.default_input_config()` → `SupportedStreamConfig`
  - `device.build_input_stream()` → `cpal::Stream` (must keep alive while recording)
  - The Stream is `!Send` in some cpal versions — keep it in the same thread that created it.

- **CRITICAL — Root/PipeWire issue (Pitfall 1 from RESEARCH.md):**
  `aucc-ui` runs as root via `pkexec`. PipeWire socket lives at `/run/user/$UID/`. Must set `XDG_RUNTIME_DIR=/run/user/$PKEXEC_UID` BEFORE calling any cpal function. The env var `PKEXEC_UID` is always set by pkexec (verified).

- **Architecture (from RESEARCH.md Pattern 1):**
  ```
  TUI thread ──AudioCmd──→ Audio engine thread
                              ├── cpal callback writes to Arc<Mutex<BandAmplitudes>>
                              └── LED drive sub-thread reads bands at 30fps
                                     └── sends UsbCmd to existing USB worker
  ```

- **LED drive loop constraints (Pitfalls 6 & 7):**
  - Cap at 30fps (33ms frame budget)
  - `apply_mono_color` = 10 USB transfers ≈ 15ms
  - `set_brightness` = 1 USB transfer ≈ 1.5ms
  - Use `LedOutput::Brightness` for Pulse effect (fast path)
  - NEVER use `save: true` for audio sync (EEPROM wear)
  - Send both keyboard + lightbar commands per frame

- **UsbCmd reuse:** The LED drive loop needs a `mpsc::Sender<UsbCmd>` clone. This is passed from `run()` in Plan 03. For now, `spawn_audio_engine()` takes it as a parameter.

- **Existing UsbCmd enum** (in `src/ui/tui.rs`, line ~92):
  ```rust
  enum UsbCmd {
      MonoColor { r, g, b, brightness, save },
      AltColor { ... },
      Effect([u8; 8]),
      Disable,
      LbColor { path, r, g, b, brightness },
      LbDisable { path },
  }
  ```
  Plan 03 will add `AudioColor` and `AudioBrightness` variants. For now, `spawn_audio_engine` is generic over a callback/sender trait — it calls a closure that the TUI wiring (Plan 03) will provide.

**1. `setup_audio_env_for_root()`** — Must be called before ANY cpal function:
```rust
use cpal::traits::{DeviceTrait, HostTrait, StreamTrait};
use std::sync::{Arc, Mutex};
use crate::audio::{BandAmplitudes, fft};

/// Set XDG_RUNTIME_DIR so cpal's ALSA backend can reach PipeWire.
/// Only modifies env when running as root (euid == 0) and PKEXEC_UID is set.
///
/// # Safety
/// Calls `set_var` which is unsafe in multi-threaded context.
/// MUST be called before any threads are spawned.
pub fn setup_audio_env_for_root() {
    if unsafe { libc::geteuid() } == 0 {
        if let Ok(uid_str) = std::env::var("PKEXEC_UID") {
            // Validate: PKEXEC_UID must be a valid non-negative integer
            if let Ok(uid) = uid_str.parse::<u32>() {
                let xdg = format!("/run/user/{uid}");
                // Safe: called before any audio threads exist
                unsafe {
                    std::env::set_var("XDG_RUNTIME_DIR", &xdg);
                    std::env::set_var("PIPEWIRE_RUNTIME_DIR", &xdg);
                    std::env::set_var("PULSE_SERVER",
                        format!("unix:{xdg}/pulse/native"));
                }
            }
            // If PKEXEC_UID is not a valid u32, skip silently (don't construct paths
            // from untrusted input — T-01-04 mitigation).
        }
    }
}
```

**2. `list_input_devices()`** — Returns `Vec<(String, String)>` of `(name, name)`:
```rust
/// Enumerate all input-capable audio devices.
/// Returns Vec of (device_name, display_description).
/// Call `setup_audio_env_for_root()` before this.
pub fn list_input_devices() -> Vec<(String, String)> {
    let host = cpal::default_host();
    let devices = match host.devices() {
        Ok(d) => d,
        Err(_) => return Vec::new(),
    };
    devices
        .filter_map(|dev| {
            // Only include devices that support input
            let has_input = dev.supported_input_configs()
                .map(|mut c| c.next().is_some())
                .unwrap_or(false);
            if !has_input {
                return None;
            }
            let name = dev.name().ok()?;
            Some((name.clone(), name))
        })
        .collect()
}
```

**3. `build_input_stream()`** — Creates the cpal stream:
```rust
/// Build a cpal input stream that feeds mono audio into FFT processing
/// and updates shared `BandAmplitudes`.
///
/// The returned `cpal::Stream` must be kept alive (not dropped) while
/// capturing. Dropping it stops the audio callback.
///
/// `device_name`: if None, uses default input device.
pub fn build_input_stream(
    device_name: Option<&str>,
    bands: Arc<Mutex<BandAmplitudes>>,
) -> Result<cpal::Stream, String> {
    let host = cpal::default_host();

    let device = match device_name {
        Some(name) => {
            host.devices()
                .map_err(|e| format!("Erro ao listar dispositivos: {e}"))?
                .find(|d| d.name().map(|n| n == name).unwrap_or(false))
                .ok_or_else(|| format!("Dispositivo não encontrado: {name}"))?
        }
        None => host.default_input_device()
            .ok_or_else(|| "Nenhum dispositivo de entrada padrão".to_string())?,
    };

    let config = device.default_input_config()
        .map_err(|e| format!("Erro na configuração de áudio: {e}"))?;
    let sample_rate = config.sample_rate().0;
    let channels = config.channels() as usize;

    let stream = device.build_input_stream(
        &config.into(),
        move |data: &[f32], _: &cpal::InputCallbackInfo| {
            // Mix to mono
            let mono: Vec<f32> = data.chunks(channels)
                .map(|ch| ch.iter().sum::<f32>() / channels as f32)
                .collect();
            // FFT → band extraction
            if let Some(new_bands) = fft::process_fft(&mono, sample_rate) {
                if let Ok(mut b) = bands.lock() {
                    fft::smooth_bands(&mut b, &new_bands);
                }
            }
        },
        |err| {
            // Error callback — must NOT panic (T-01-05: cpal runs this on RT thread)
            eprintln!("cpal stream error: {err}");
        },
        None, // no timeout
    ).map_err(|e| format!("Erro ao criar stream de áudio: {e}"))?;

    stream.play().map_err(|e| format!("Erro ao iniciar stream: {e}"))?;
    Ok(stream)
}
```

#### Step 1: Implement `spawn_audio_engine()` in `aucc-rs/src/audio/mod.rs`

Extend `mod.rs` with these imports and implementation below the existing types:

```rust
use std::path::PathBuf;
use std::sync::{mpsc, Arc, Mutex};
use std::thread;
use std::time::{Duration, Instant};

use crate::audio::capture;
use crate::audio::effects::AudioEffectState;
use crate::ui::tui::UsbCmd;  // UsbCmd lives in tui.rs, not keyboard/mod.rs

/// Frame budget for LED drive loop: 30fps = 33ms per frame.
pub const FRAME_BUDGET: Duration = Duration::from_millis(33);

/// Spawn the audio engine on a dedicated thread.
///
/// Architecture (Pattern 1 from RESEARCH.md):
/// - Engine thread: owns cpal Stream + processes AudioCmd from TUI
/// - LED drive sub-thread: reads BandAmplitudes at 30fps, sends UsbCmd directly
///
/// Both threads bypass the TUI main loop entirely — LED latency is ≤33ms,
/// never inflated by the 50ms TUI event poll.
///
/// # Arguments
/// - `audio_rx`: receives AudioCmd from TUI (Enable/Disable/SetEffect)
/// - `usb_tx`: direct clone of the USB worker channel — LED drive sends here
/// - `lb_path`: optional lightbar HID path for synchronized lightbar
pub fn spawn_audio_engine(
    audio_rx: mpsc::Receiver<AudioCmd>,
    usb_tx: mpsc::Sender<UsbCmd>,
    lb_path: Option<PathBuf>,
) -> thread::JoinHandle<()> {
    thread::spawn(move || {
        // MUST be called before any cpal function — sets XDG_RUNTIME_DIR for PipeWire
        capture::setup_audio_env_for_root();

        let bands: Arc<Mutex<BandAmplitudes>> = Arc::new(Mutex::new(BandAmplitudes::default()));
        let mut stream: Option<cpal::Stream> = None;
        let mut led_thread: Option<LedDriveHandle> = None;

        for cmd in audio_rx {
            match cmd {
                AudioCmd::Enable { device_name, effect } => {
                    // Stop any existing stream + LED drive before restarting
                    stream.take(); // Drop stops cpal stream
                    if let Some(handle) = led_thread.take() {
                        handle.stop();
                    }

                    // Reset band state
                    if let Ok(mut b) = bands.lock() {
                        *b = BandAmplitudes::default();
                    }

                    match capture::build_input_stream(
                        device_name.as_deref(),
                        Arc::clone(&bands),
                    ) {
                        Ok(s) => {
                            stream = Some(s);

                            // Spawn LED drive sub-thread with direct usb_tx clone
                            led_thread = Some(LedDriveHandle::spawn(
                                Arc::clone(&bands),
                                effect,
                                usb_tx.clone(),
                                lb_path.clone(),
                            ));
                        }
                        Err(e) => {
                            eprintln!("Audio engine: failed to start capture: {e}");
                        }
                    }
                }
                AudioCmd::Disable => {
                    stream.take();
                    if let Some(handle) = led_thread.take() {
                        handle.stop();
                    }
                }
                AudioCmd::SetEffect(effect) => {
                    if let Some(ref handle) = led_thread {
                        handle.set_effect(effect);
                    }
                }
            }
        }
        // Channel closed — engine shutting down
        stream.take();
        if let Some(handle) = led_thread.take() {
            handle.stop();
        }
    })
}

/// Handle to the LED drive sub-thread.
/// Owns the stop and effect change channels.
struct LedDriveHandle {
    stop_tx: mpsc::Sender<()>,
    effect_tx: mpsc::Sender<effects::AudioEffect>,
    thread: Option<thread::JoinHandle<()>>,
}

impl LedDriveHandle {
    fn spawn(
        bands: Arc<Mutex<BandAmplitudes>>,
        initial_effect: effects::AudioEffect,
        usb_tx: mpsc::Sender<UsbCmd>,
        lb_path: Option<PathBuf>,
    ) -> Self {
        let (stop_tx, stop_rx) = mpsc::channel::<()>();
        let (effect_tx, effect_rx) = mpsc::channel::<effects::AudioEffect>();

        let thread = thread::spawn(move || {
            let mut effect_state = AudioEffectState::new(initial_effect);

            loop {
                let frame_start = Instant::now();

                if stop_rx.try_recv().is_ok() {
                    break;
                }

                if let Ok(new_effect) = effect_rx.try_recv() {
                    effect_state = AudioEffectState::new(new_effect);
                }

                let bands_snapshot = bands.lock()
                    .map(|b| b.clone())
                    .unwrap_or_default();

                effect_state.tick(FRAME_BUDGET.as_secs_f32());
                let output = effect_state.render(&bands_snapshot);

                // Send directly to USB worker — NO TUI routing
                match output {
                    LedOutput::Brightness(level) => {
                        // try_send: if channel full, drop this frame (backpressure)
                        // send(): blocking only if USB worker thread is dead (unlikely at 30fps)
                    let _ = usb_tx.send(UsbCmd::AudioBrightness(level));
                    }
                    LedOutput::Color { r, g, b, brightness } => {
                        let _ = usb_tx.send(UsbCmd::AudioColor { r, g, b, brightness });

                        // Synchronize lightbar if available
                        if let Some(ref path) = lb_path {
                            use crate::lightbar;
                            let _ = lightbar::apply_color(path, r, g, b, brightness);
                        }
                    }
                }

                // Maintain ≤33ms frame budget (30fps)
                let elapsed = frame_start.elapsed();
                if elapsed < FRAME_BUDGET {
                    thread::sleep(FRAME_BUDGET - elapsed);
                }
            }
        });

        Self {
            stop_tx,
            effect_tx,
            thread: Some(thread),
        }
    }

    fn stop(mut self) {
        let _ = self.stop_tx.send(());
        if let Some(t) = self.thread.take() {
            let _ = t.join();
        }
    }

    fn set_effect(&self, effect: effects::AudioEffect) {
        let _ = self.effect_tx.send(effect);
    }
}
```

**Note on `LedOutput`:** The `LedOutput` enum (defined in Plan 01's `mod.rs`) remains as an
internal type used by `effect_state.render()`. It is converted to `UsbCmd` variants inside
the LED drive loop — it is **never** exposed through a channel to the TUI.

## Threat Model

| Boundary | Description |
|----------|-------------|
| Environment → cpal | `PKEXEC_UID` from pkexec env; `XDG_RUNTIME_DIR` constructed from it |
| cpal callback → FFT | Raw f32 audio samples from ALSA |
| Audio engine → UsbCmd | LED output channel (internal) |

| Threat ID | Category | Component | Disposition | Mitigation |
|-----------|----------|-----------|-------------|------------|
| T-01-04 | Elevation of Privilege | `setup_audio_env_for_root` | mitigate | Validate `PKEXEC_UID` is a valid `u32` before constructing path; don't pass arbitrary strings to env |
| T-01-05 | Denial of Service | cpal error callback | mitigate | Error callback uses `eprintln!` only, never panics; crash of RT audio thread would bring down root process |
| T-01-06 | Tampering | device_name parameter | mitigate | Device name is matched against `device.name()` from cpal enumeration — no shell injection possible; cpal validates internally |
| T-01-07 | Denial of Service | LED output channel | mitigate | `usb_tx` is an unbounded `mpsc::channel`. At 30fps the LED drive sends at most 30 msgs/sec; the USB worker processes each in ~15ms and keeps up. No backpressure needed at this rate. |
