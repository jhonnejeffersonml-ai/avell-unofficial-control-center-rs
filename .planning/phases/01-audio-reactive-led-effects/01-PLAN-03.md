# Plan 03: TUI Integration — Audio Sync Screen & Wiring

**Phase:** 01 — Audio-reactive LED effects
**Complexity:** High
**Depends on:** Plan 02 (needs complete `audio` module: `spawn_audio_engine`, `AudioCmd`, `LedOutput`, `AudioEffect`, `list_input_devices`)
**Wave:** 3

## Objective

Wire the audio engine into the existing TUI: extend `UsbCmd` with audio variants, add `Screen::AudioSync` and `Screen::AudioDevice` screens, implement LED snapshot/restore, and connect everything so the user can enable/disable audio sync from the menu.

## Context

- **Primary file to modify:** `aucc-rs/src/ui/tui.rs` (~1500 lines). This is the only file with production changes in this plan.

- **Existing patterns to follow exactly:**
  - Screen navigation: `Screen` enum → `item_count()` → `confirm()` → `go_back()` → `screen_title()` → `build_list_items()` → `render()`
  - USB command dispatch: `UsbCmd` enum → `spawn_usb_worker()` match arms
  - Main menu items: `main_items_dynamic()` returns `Vec<&'static str>`
  - Live preview: `live_preview()` sends `UsbCmd` on cursor move
  - Confirmation: `confirm()` matches `self.screen` and handles selection

- **Key interfaces from Plan 01-02 (audio module):**

- **UsbCmd extension (from RESEARCH.md):**
  Add two new variants:

- **LED snapshot/restore (Success Criterion 6):**
  Before enabling audio sync, capture current `(color_a → r,g,b, brightness)` from AppState.
  On disable, restore by sending the saved `UsbCmd::MonoColor`.

- **Audio device list:**
  Populated lazily when user enters `Screen::AudioSync` for the first time.
  Calls `audio::capture::list_input_devices()` (which needs `setup_audio_env_for_root()` called first — done by `spawn_audio_engine()`).

- **User-facing labels in Portuguese** (following existing TUI convention):
  - Menu item: `"🎵 Audio Sync"`
  - Screen title: `"🎵 Audio Sync — LEDs reagem ao áudio"`
  - Device picker title: `"🎵 Selecionar fonte de áudio"`
  - Status messages: `"Audio sync ativado!"`, `"Audio sync desativado."`

**7. Implement `enable_audio_sync()` and `disable_audio_sync()` on AppState:**
```rust
fn enable_audio_sync(&mut self) {
    // Save current LED state for restore on disable
    let (r, g, b) = color_at(self.color_a);
    self.audio_snapshot = Some(LedSnapshot {
        r, g, b,
        brightness: self.brightness,
    });

    // Create audio command channel
    let (audio_tx, audio_rx) = mpsc::channel::<AudioCmd>();

    // Create LED output channel (bounded, size 2 — backpressure)
    let (led_tx, led_rx) = mpsc::sync_channel::<LedOutput>(2);

    // Spawn audio engine
    let lb_path = self.lb_path.clone();
    audio::spawn_audio_engine(audio_rx, led_tx, lb_path);

    // Send Enable command with current settings
    let device_name = self.audio_device_idx
        .and_then(|idx| self.audio_devices.get(idx))
        .map(|(name, _)| name.clone());

    let _ = audio_tx.send(AudioCmd::Enable {
        device_name,
        effect: self.audio_effect,
    });

    self.audio_tx = Some(audio_tx);
    self.led_rx = Some(led_rx);
    self.audio_enabled = true;
    self.status = "🎵 Audio sync ativado!".into();
}

fn disable_audio_sync(&mut self) {
    // Send Disable to audio engine
    if let Some(ref tx) = self.audio_tx {
        let _ = tx.send(AudioCmd::Disable);
    }
    self.audio_tx = None;
    self.led_rx = None;
    self.audio_enabled = false;

    // Restore previous LED state
    if let Some(snap) = self.audio_snapshot.take() {
        let _ = self.usb_tx.send(UsbCmd::MonoColor {
            r: snap.r, g: snap.g, b: snap.b,
            brightness: snap.brightness, save: false,
        });
        // Also restore lightbar
        if let Some(path) = self.lb_path.clone() {
            let _ = self.usb_tx.send(UsbCmd::LbColor {
                path, r: snap.r, g: snap.g, b: snap.b,
                brightness: 0x32, // default lightbar brightness
            });
        }
    }

    self.status = "Audio sync desativado.".into();
}
```

## Threat Model

| Boundary | Description |
|----------|-------------|
| TUI → Audio engine | AudioCmd over mpsc channel |
| Audio engine → USB worker | LedOutput → UsbCmd forwarding in main loop |
| User input → device selection | Index into pre-enumerated device list |

| Threat ID | Category | Component | Disposition | Mitigation |
|-----------|----------|-----------|-------------|------------|
| T-01-08 | Tampering | Device selection | mitigate | User selects by index into `audio_devices` Vec (pre-enumerated by cpal); no raw string input accepted |
| T-01-09 | Denial of Service | EEPROM wear | mitigate | All `AudioColor` and `AudioBrightness` commands use `save: false`; enforced in `spawn_usb_worker` match arm |
| T-01-10 | Information Disclosure | Audio data | accept | Audio samples are processed in-memory only (FFT → bands); raw audio is never logged, saved, or transmitted. cpal callback data lives in stack/heap of the audio thread only |
| T-01-11 | Denial of Service | Channel overflow | mitigate | `led_tx` is `sync_channel(2)`; `try_send` drops stale frames. `audio_tx` is unbounded but only receives 3 command types from user interaction (not automated) |
